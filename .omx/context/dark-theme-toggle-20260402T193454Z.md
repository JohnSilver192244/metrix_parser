# Context Snapshot

- Task statement: Add a dark theme and a theme toggle to the web app.
- Desired outcome: The whole web app supports light and dark themes, with a manual switch in the top-right corner.
- Stated solution: Use manual theme switching only; do not persist the user’s choice between sessions or reloads.
- Probable intent hypothesis: Improve visual comfort and modernize the app’s UI without changing layout or behavior.
- Known facts/evidence:
  - Repository is a brownfield monorepo.
  - `apps/web` is a React/Vite app.
  - `apps/web/src/styles/global.css` currently sets `color-scheme: light;`.
  - The user wants the theme applied to the entire web app.
  - The user wants the default theme to be light on each fresh load.
  - The user wants colors based on Material 3 guidance.
- Constraints:
  - Do not change layout.
  - Change only colors, contrast, and states.
  - No persistence across reloads or sessions.
- Unknowns/open questions:
  - Which implementation approach best fits the existing React/Vite structure.
  - Whether the toggle should be a button, switch, or icon control.
- Decision-boundary unknowns:
  - What OMX may decide without confirmation: exact component implementation, theme token structure, and DOM/CSS wiring.
- Likely codebase touchpoints:
  - `apps/web/src/styles/global.css`
  - `apps/web/src/**` theme-related UI and shared styling
