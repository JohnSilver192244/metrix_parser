# PRD: settings-unified-sequence

## Цель
Собрать запуск обновлений в единый последовательный UI-процесс без изменений API и бизнес-алгоритмов.

## Требования
1. Интерфейс запуска обновлений должен быть последовательным (шаги: период -> опции -> запуск).
2. Левое меню шагов должно быть фиксированным (sticky) на десктопе.
3. Запуск любого сценария должен быть заблокирован, пока не выбран диапазон дат.
4. В `UpdatePeriodPicker` максимальная ширина диапазона: 14 дней.
5. Текстовый контракт: использовать строку `всего строк` в соответствующем UI.
6. Изменения только в UI/верстке/копирайте; API и алгоритмы не трогаем.

## Критерии приемки
- На странице обновлений видны шаги и sticky-левое меню.
- Кнопки запуска disabled без диапазона.
- При `maxRangeDays=14` диапазон >14 дней не применяется.
- Пройден регресс web-тестов и typecheck.
